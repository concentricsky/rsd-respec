This document describes the Rich Skill Descriptor (RSD) class and how RSDs should be published for compatibility with services that process machine readable representations of skills and competencies. Each RSD is primarily identified by a canonical URL and consists of a skillStatement and supporting metadata to identify alignment to other skills or job categories.

Introduction

In order to create a ecosystem of recognition around skills, where Achievements, Pathways, and Learner Records make machine-readable references to skills and allow systems to react to the skills learners hold, it is important for implementers to use skills in common ways. RSDs build on CTDL-ASN to enable skill definers to offer definitions that can be referenced from digital credentials (including those that appear in learner records), pathways, and job profiles.

RSD Data Model

RSD Property Field Name Data Type Description CTDL-ASN mapping
id ID URI Canonical URL that uniquely identifies the skill, from which more information may be retrieved. @id
skillName Skill Name String Short name or label for the skill. Required ctdlasn:competencyLabel
skillStatement Skill Statement SkillStatement Skills description written in standard syntax. Required ctdlasn:competencyText
keywords Keywords Array[String] List of keyword strings to enhance searchability. ctdl:keyword
category Category String A category in which this RSD is filed, in the local vocabulary defined by the issuer ctdlasn:competencyCategory
author Author URI A person or organization chiefly responsible for the intellectual or artistic content of this skill ctdlasn:author
alignments Alignments Array[String:URI] indicates alignment or overlap of this RSD to an underlying unique identifiers of standards, skills, or competencies identified in other frameworks or data sources (i.e.,Emsi; Burning Glass; LinkedIn; Workday) ctdlasn:majorAlignment *
certifications Certifications Array[String:URI] Field for associating external certifications to RSDs ctdlasn:alignFrom
occupations Occuption Data Categories Array[String:JobCode] indicates skill alignment to releveant job roles/occupations aligned to an external categorization like BLS SOC or O*NET ctdl:occupationType
employers Employers Array[String] indicates support for a skill by specific employers who employ people who hold the skill ctdl:employementOutcome
skillEmbodied Source Skill Embodied Array[String] indicates alignment of this skill to data sources or keywords that do not have canonical URIs ctdlasn:skillEmbodied

Available alignment types include majorAlignment, broadAlignment, exactAlignment, minorAlignment, narrowAlignment, and prerequisiteAlignment. The most simple or default recommendation is majorAlignment, which implies a close but not exact match.

Skill Statement

The structured SkillStatement allows for skill statements to have a consistent syntax. The full skill statement will be made up by concatenating all parts together, e.g.: "{Subject} {Predicate} {Statement}".

RSD Property Field Type Description
subject Subject String Who this skill applies to, e.g "Student"
predicate Predicate String
statement Skill Statement String The body of the skill statement

Example RSD

{
 "@context": "https://rsd.osmt.dev/context-v1.json",
 "id": "https://osmt.example.com/skills/9cd84455-2086-4f41-bcac-6d518beeb6ac",
 "type": "RichSkillDescriptor",
 "author": "https://osmt.example.com/organization",
 "skillName": "Document Saving",
 "skillStatement": "The learner copies saves documents to their local machine.",
 "alignments": ["https://skillsource.example.com/base-skills/simple-compuhacking"],
 "keywords": ["documents", "file operations", "computer basics"],
 "category": "Productivity Software",
 "occupations": ["13-0000", "13-2000", "13-2010", "13-2030", "13-2040", "13-2060", "13-2080"]
}
