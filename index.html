This document describes the Rich Skill Descriptor (RSD) class and how RSDs should be published for compatibility with services that process machine readable representations of skills and competencies. Each RSD is primarily identified by a canonical URL and consists of a skillStatement and supporting metadata to identify alignment to other skills or job categories.

Introduction

In order to create a ecosystem of recognition around skills, where Achievements, Pathways, and Learner Records make machine-readable references to skills and allow systems to react to the skills learners hold, it is important for implementers to use skills in common ways. RSDs build on CTDL-ASN to enable skill definers to offer definitions that can be referenced from digital credentials (including those that appear in learner records), pathways, and job profiles.

RSD Data Model

RSD Property Field Type Description CTDL-ASN mapping
id ID URI Canonical URL that uniquely identifies the skill, from which more information may be retrieved. identifier
name Skill Name String Short name or label for the skill competencyLabel
skillStatement Skill Statement String Skills description written in standard syntax competencyText
skillFamily Skill Family String The textual label identifying the category of the competency as designated by the promulgating body. competencyCategory
author Author URI A person or organization chiefly responsible for the intellectual or artistic content of this skill author
frameworks Framework(s) Array[String] Align to a skill defined in another framework alignment

industry Industry String SOC code of the applicable industry industryType
jobFunctions Job Function(s) Array[String] SOC codes of the applicabile job functions occupationType
jobRoles Job Role(s) Array[String] SOC codes of applicable job roles occupationType
occupations Occupation(s) Array[String] SOC codes of the applicable occupations occupationType
